Methodology

State the model. Expose the controls. Make the run legible.

Vibe Simulation is not built around a black-box wow moment. It is built around making numerical behavior, runtime constraints, and interaction choices intelligible to the user.

Method pillars
  • Determinism where it matters. Seeded behavior and versioned scenes support reproducibility.
  • Performance honesty. Heavy compute belongs in dedicated lab pages, not hidden in the landing surface.
  • Evidence over screenshots. Receipts and verifiers create a better review trail than static captures.
Public claims
Claim How the site supports it
Reproducibility Deterministic seeds, stable scene references, and explicit run metadata where available.
Verifiability Reasoning Reels and the standalone verifier create a portable integrity layer.
Performance awareness Heavy labs are isolated from the homepage so the public shell remains fast and readable.
Pedagogical clarity Controls are paired with outcome framing instead of being left as unexplained sliders.
What this does not claim
  • Not certification. Verification features support review and reproducibility, not formal accreditation by themselves.
  • Not infinite fidelity. These are browser labs, so each model carries assumptions and runtime tradeoffs.
  • Not generic edtech filler. The site is moving away from thin content and toward strong, inspectable lab surfaces.
